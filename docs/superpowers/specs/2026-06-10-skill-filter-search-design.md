# Skill 管理页面 — 搜索过滤与排序

## 目标

为 Skill 管理页面（`SkillSettings.vue`）增加按来源、名称、描述搜索过滤的功能，并按名称排序呈现。

## 方案：纯前端过滤

Skill 数据量小（几十条），前端已有全量数据，直接在组件内做本地过滤 + 排序，无需后端改动。

## 设计

### 过滤栏 UI

在页面 `subtitle` 和 `skill-list` 之间新增一行过滤栏：

- **关键词搜索框**（左侧）— 输入文本，模糊匹配 `skillName`、`displayName`、`description`（大小写不敏感）
- **来源下拉筛选**（右侧）— `全部来源 | 内置 | Git | URL | 本地 | 上传 | 同步 | 市场`
- **清除按钮** — 有筛选条件时显示，一键重置
- **计数文本** — 如 `(5/11)`，显示过滤后/总数

布局：搜索框 + 来源下拉 + 清除按钮 + 计数，同一行水平排列，背景用 `surface-soft` 圆角容器包裹。

### 过滤与排序逻辑

新增 computed 属性 `filteredSkills`：

```
store.skills
  → filter by keyword (skillName/displayName/description 任一包含)
  → filter by source (选中非"全部"时匹配)
  → sort by skillName (localeCompare, 升序)
```

模板中 `v-for="s in store.skills"` 改为 `v-for="s in filteredSkills"`。

### 空结果状态

当过滤后无结果时显示空状态提示"没有匹配的 Skill"，与初始安装前的空状态区分。

### 非目标

- 不做后端接口改动
- 不做数据库查询改动
- 不新增排序方向切换（仅升序）

## 影响范围

| 层 | 文件 | 变更 |
|---|---|---|
| 前端 | `SkillSettings.vue` | 新增过滤栏 UI + computed + 空结果状态 |
| 后端 | — | 无变更 |
| Store | — | 无变更 |
| Mapper | — | 无变更 |
